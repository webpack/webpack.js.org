للتمييز داخل `webpack.config.js` بين [التطوير](/guides/development) و[بناءات الإنتاج](/guides/production)، يمكنك استخدام متغيرات البيئة.

T> تختلف متغيرات البيئة الخاصة بـ webpack عن [متغيرات البيئة](https://en.wikipedia.org/wiki/Environment_variable) الخاصة بأصداف أنظمة التشغيل مثل `bash` و`CMD.exe`.

يسمح لك [خيار البيئة](/api/cli/#environment-options) في سطر أوامر webpack، وهو `--env`، بتمرير أي عدد تريده من متغيرات البيئة. وستصبح هذه المتغيرات متاحة داخل `webpack.config.js`. على سبيل المثال: `--env production` أو `--env goal=local`.

```bash
npx webpack --env goal=local --env production --progress
```

T> عند إعداد متغير `env` دون إسناد قيمة، فإن `--env production` يعيّن `env.production` إلى `true` افتراضيًا. توجد أيضًا صيغ أخرى يمكنك استخدامها. راجع توثيق [webpack CLI](/api/cli/#environment-options) لمزيد من المعلومات.

هناك تغيير واحد ستحتاج إلى إجرائه في إعدادات webpack. عادةً يشير `export default` إلى كائن الإعدادات. لاستخدام متغير `env`، يجب أن تجعل `export default` دالة:

**webpack.config.js**

```js
import path from "node:path";
import { fileURLToPath } from "node:url";

const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);

export default (env) => {
  // استخدم env.<YOUR VARIABLE> هنا:
  console.log("Goal:", env.goal); // 'local'
  console.log("Production:", env.production); // true

  return {
    entry: "./src/index.js",
    output: {
      filename: "bundle.js",
      path: path.resolve(__dirname, "dist"),
    },
  };
};
```

T> يوفّر Webpack CLI بعض [متغيرات البيئة المدمجة](/api/cli/#cli-environment-variables) التي يمكنك الوصول إليها داخل إعدادات webpack.
