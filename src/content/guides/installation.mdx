يستعرض هذا الدليل الطرق المختلفة المستخدمة لتثبيت webpack.

## المتطلبات المسبقة

قبل أن نبدأ، تأكد من تثبيت إصدار حديث من [Node.js](https://nodejs.org/en/). يُعد إصدار الدعم طويل الأمد (LTS) الحالي نقطة بداية مناسبة. قد تواجه مشكلات مختلفة مع الإصدارات الأقدم، لأنها قد تفتقد وظائف يحتاج إليها webpack و/أو الحزم المرتبطة به.

## التثبيت المحلي

<span id="local-installation"></span>

أحدث إصدار من webpack هو:

[![GitHub release](https://img.shields.io/npm/v/webpack.svg?label=webpack&style=flat-square&maxAge=3600)](https://github.com/webpack/webpack/releases)

لتثبيت أحدث إصدار أو إصدار محدد، نفّذ أحد الأوامر التالية:

```bash
npm install --save-dev webpack
# أو إصدار محدد
npm install --save-dev webpack@<version>
```

T> يعتمد استخدام `--save-dev` من عدمه على حالة الاستخدام لديك. إذا كنت تستخدم webpack للتجميع فقط، فمن المقترح تثبيته باستخدام الخيار `--save-dev` لأنك لن تضمّن webpack داخل بناء الإنتاج. بخلاف ذلك يمكنك تجاهل `--save-dev`.

إذا كنت تستخدم webpack v4 أو أحدث وتريد استدعاء `webpack` من سطر الأوامر، فستحتاج أيضًا إلى تثبيت [CLI](/api/cli/).

```bash
npm install --save-dev webpack-cli
```

التثبيت المحلي هو ما نوصي به لمعظم المشاريع. فهذا يجعل ترقية المشاريع كلٌّ على حدة أسهل عند إدخال تغييرات كاسرة. عادةً يتم تشغيل webpack عبر واحد أو أكثر من [npm scripts](https://docs.npmjs.com/misc/scripts)، وهي ستبحث عن تثبيت webpack داخل مجلد `node_modules` المحلي:

```json
"scripts": {
  "build": "webpack --config webpack.config.js"
}
```

T> لتشغيل التثبيت المحلي من webpack يمكنك الوصول إلى الملف التنفيذي الخاص به عبر `node_modules/.bin/webpack`. وبدلًا من ذلك، إذا كنت تستخدم npm v5.2.0 أو أحدث، يمكنك تنفيذ `npx webpack`.

## التثبيت العام

سيجعل تثبيت NPM التالي الأمر `webpack` متاحًا على مستوى النظام:

```bash
npm install --global webpack
```

W> لاحظ أن هذه **ليست ممارسة موصى بها**. التثبيت العام يقيّدك بإصدار محدد من webpack وقد يفشل في المشاريع التي تستخدم إصدارًا مختلفًا.

## الإصدارات الأحدث قيد التطوير

إذا كنت متحمسًا لاستخدام أحدث ما يقدمه webpack، يمكنك تثبيت الإصدارات التجريبية أو التثبيت مباشرةً من مستودع webpack باستخدام الأوامر التالية:

```bash
npm install --save-dev webpack@next
# أو وسم/فرع محدد
npm install --save-dev webpack/webpack#<tagname/branchname>
```

W> توخَّ الحذر عند تثبيت هذه الإصدارات الأحدث قيد التطوير! فقد تظل تحتوي على أخطاء، ولذلك لا ينبغي استخدامها في الإنتاج.
