يُعد `EnvironmentPlugin` اختصارًا لاستخدام [`DefinePlugin`](/plugins/define-plugin) مع مفاتيح [`process.env`](https://nodejs.org/api/process.html#process_process_env).

## الاستخدام

يقبل `EnvironmentPlugin` إما array من المفاتيح، أو object يربط كل مفتاح بقيمته الافتراضية.

```js
new webpack.EnvironmentPlugin(["NODE_ENV", "DEBUG"]);
```

هذا يعادل استخدام `DefinePlugin` بالشكل التالي:

```js
new webpack.DefinePlugin({
  "process.env.NODE_ENV": JSON.stringify(process.env.NODE_ENV),
  "process.env.DEBUG": JSON.stringify(process.env.DEBUG),
});
```

W> يبحث `EnvironmentPlugin` عن المتغير المحدد داخل `process.env`. إذا لم يجده، يبحث عن قيمة افتراضية داخل الإعدادات. إذا لم يجد لا المتغير ولا القيمة الافتراضية، فسيظهر خطأ: **"`EnvironmentPlugin` - `${key}` environment variable is `undefined`"**.

## الاستخدام مع القيم الافتراضية

يمكنك أيضًا تمرير object إلى `EnvironmentPlugin` يربط المفاتيح بقيمها الافتراضية. تُستخدم القيمة الافتراضية إذا كان المفتاح غير موجود في `process.env`.

```js
new webpack.EnvironmentPlugin({
  NODE_ENV: "development", // use 'development' unless process.env.NODE_ENV is defined
  DEBUG: false,
});
```

W> القيم القادمة من `process.env` تكون دائمًا strings.

T> بخلاف [`DefinePlugin`](/plugins/define-plugin)، يطبّق `EnvironmentPlugin` دالة `JSON.stringify` على القيم الافتراضية.

T> القيمتان الافتراضيتان `null` و`undefined` لا تتصرفان بالطريقة نفسها. استخدم `undefined` للمتغيرات التي يجب توفيرها أثناء bundling، واستخدم `null` إذا كانت اختيارية.

**مثال:**

لنرَ ماذا يحدث عند تشغيل إعداد `EnvironmentPlugin` السابق على ملف اختبار اسمه `entry.js`:

```js
if (process.env.NODE_ENV === "production") {
  console.log("Welcome to production");
}
if (process.env.DEBUG) {
  console.log("Debugging output");
}
```

عند تنفيذ `NODE_ENV=production webpack` في الطرفية للبناء، يتحول `entry.js` إلى الشكل التالي:

{/* eslint-disable no-self-compare, no-constant-condition */}

```js
if ("production" === "production") {
  // <-- 'production' from NODE_ENV is taken
  console.log("Welcome to production");
}
if (false) {
  // <-- default value is taken
  console.log("Debugging output");
}
```

أما تشغيل `DEBUG=false webpack` فينتج عنه:

{/* eslint-disable no-self-compare, no-constant-condition */}

```js
if ("development" === "production") {
  // <-- default value is taken
  console.log("Welcome to production");
}
if ("false") {
  // <-- 'false' from DEBUG is taken
  console.log("Debugging output");
}
```

## حالة استخدام: إصدار Git

يوفّر إعداد `EnvironmentPlugin` التالي القيمتين `process.env.GIT_VERSION`، مثل "v5.4.0-2-g25139f57f"، و`process.env.GIT_AUTHOR_DATE`، مثل "2020-11-04T12:25:16+01:00"، بناءً على آخر commit في مستودع Git:

```js
import childProcess from "node:child_process";

function git(command) {
  return childProcess.execSync(`git ${command}`, { encoding: "utf8" }).trim();
}

new webpack.EnvironmentPlugin({
  GIT_VERSION: git("describe --always"),
  GIT_AUTHOR_DATE: git("log -1 --format=%aI"),
});
```

## DotenvPlugin

يسمح plugin خارجي وهو [`DotenvPlugin`](https://github.com/mrsteele/dotenv-webpack) (`dotenv-webpack`) بكشف جزء من [متغيرات dotenv](https://www.npmjs.com/package/dotenv) أو كلها:

```bash
// .env
DB_HOST=127.0.0.1
DB_PASS=foobar
S3_API=mysecretkey
```

```js
new Dotenv({
  path: "./.env", // Path to .env file (this is the default)
  safe: true, // load .env.example (defaults to "false" which does not use dotenv-safe)
});
```
