يسهّل [`HtmlWebpackPlugin`](https://github.com/jantimon/html-webpack-plugin) إنشاء ملفات HTML التي تحمل bundles الناتجة من webpack. هذا مفيد خصوصًا عندما تحتوي أسماء ملفات bundles على hash يتغير مع كل عملية بناء. يمكنك ترك plugin ينشئ ملف HTML لك، أو تمرير قالب خاص بك باستخدام [lodash templates](https://lodash.com/docs#template)، أو استخدام [loader](/loaders) من اختيارك.

## التثبيت

```bash
npm install --save-dev html-webpack-plugin
```

## الاستخدام الأساسي

ينشئ plugin ملف HTML5 يحتوي كل webpack bundles داخل عنصر `body` باستخدام وسوم `script`. أضف plugin إلى إعدادات webpack بهذه الطريقة:

```js
import path from "node:path";
import { fileURLToPath } from "node:url";
import HtmlWebpackPlugin from "html-webpack-plugin";

const __filename = fileURLToPath(import.meta.url);
const __dirname = path.dirname(__filename);

export default {
  entry: "index.js",
  output: {
    path: path.resolve(__dirname, "./dist"),
    filename: "index_bundle.js",
  },
  plugins: [new HtmlWebpackPlugin()],
};
```

سينشئ ذلك ملف `dist/index.html` بالمحتوى التالي:

```html
<!DOCTYPE html>
<html>
  <head>
    <meta charset="UTF-8" />
    <title>webpack App</title>
  </head>
  <body>
    <script src="index_bundle.js"></script>
  </body>
</html>
```

إذا كان لديك أكثر من webpack entry point، فستُضاف كلها إلى ملف HTML الناتج باستخدام وسوم `<script>`.

إذا كانت مخرجات webpack تحتوي CSS assets، مثل CSS المستخرج باستخدام [MiniCssExtractPlugin](/plugins/mini-css-extract-plugin/)، فستُضاف هذه الملفات إلى عنصر `<head>` في HTML الناتج باستخدام وسوم `<link>`.
