عندما تقوم بكتابة الكود، من المحتمل ان تكون بالفعل اضفت الكثير من نقاط الافتراق لتحميل اشياء حسب الحاجة. بعد جمع الكود (compile) ستلاحظ ان بعض الاجزاء صغيرة للغاية - مما ينشئ طلبات HTTP اكبر. 
`LimitChunkCountPlugin` يمكنه معالجة الطلبات بشكل مسبق عن طريق دمج الاجزاء معاً

```js
new webpack.optimize.LimitChunkCountPlugin({
  // خيارات...
});
```

## الخيارات

الخيارات ادناه هي المدعومة:

### maxChunks

`رقم`

تحديد عدد اقصى من الاجزاء بأستعمال قيمة اكبر او تساوي `1` استعمال `1` يمنع اي اجزاء اضافية من اضافتها كأجزاء/مداخل رئيسية وكذلك هي محسوبة في العدد

**webpack.config.js**

```js
import webpack from "webpack";

export default {
  // ...
  plugins: [
    new webpack.optimize.LimitChunkCountPlugin({
      maxChunks: 5,
    }),
  ],
};
```

### minChunkSize

إبقاء عدد الاجزاء فوق الحد المسموح لم يعد ميزة خاصة بهذه الإضافة. استعمل [إضافة الحد الأدنى للاجزاء](/plugins/min-chunk-size-plugin) بدلاً عنها

## الاستعمال عن طريق واجهة الأوامر (CLI)

هذه الإضافة وخياراتها يمكن استعمالها عن طريق واجهة الاوامر [CLI](/api/cli/):

```bash
webpack --optimize-max-chunks 15
```
